The DepthAI Pipeline Mental Model

The one concept that makes everything else make sense

A DepthAI pipeline is a directed graph of nodes that runs on the device. You build the graph on the host (your PC), upload it, and the OAK executes it. Your host code just reads named output queues.

The lifecycle in three words: Build → Upload → Consume.
Steps 1–5 in any DepthAI program describe a graph. Nothing runs until dai::Device device(pipeline). After that, your host only ever reads queues.

This is why demos looked like magic — there's a clean boundary between graph construction (on your machine) and graph execution (on the OAK's VPU/ISP). Once you see that boundary, everything falls into place.

Annotated C++ — Building a Pipeline

Every line maps to something in the graph above

// 1. Create the pipeline — just a description, nothing runs yet
dai::Pipeline pipeline;

// 2. Create nodes — objects in the graph
auto camRgb    = pipeline.create<dai::node::ColorCamera>();
auto monoLeft  = pipeline.create<dai::node::MonoCamera>();
auto monoRight = pipeline.create<dai::node::MonoCamera>();
auto stereo    = pipeline.create<dai::node::StereoDepth>();
auto xoutRgb   = pipeline.create<dai::node::XLinkOut>();
auto xoutDepth = pipeline.create<dai::node::XLinkOut>();

// 3. Configure nodes — set properties BEFORE linking
camRgb->setResolution(dai::ColorCameraProperties::SensorResolution::THE_1080_P);
monoLeft->setResolution(dai::MonoCameraProperties::SensorResolution::THE_400_P);
monoLeft->setCamera("left");    // physical camera socket
monoRight->setResolution(dai::MonoCameraProperties::SensorResolution::THE_400_P);
monoRight->setCamera("right");
stereo->setDefaultProfilePreset(dai::node::StereoDepth::PresetMode::HIGH_DENSITY);

// 4. Name the output queues — how you retrieve data on the host
xoutRgb->setStreamName("rgb");
xoutDepth->setStreamName("depth");

// 5. Link the ports — THIS is the graph wiring
//    always: source.outputPort.link(destination.inputPort)
monoLeft->out.link(stereo->left);
monoRight->out.link(stereo->right);
camRgb->video.link(xoutRgb->input);
stereo->depth.link(xoutDepth->input);

// 6. Start the device — NOW it uploads the graph and runs on the OAK
dai::Device device(pipeline);

// 7. Get host-side queue handles — your only touch points from here on
auto rgbQueue   = device.getOutputQueue("rgb",   4, false); // maxSize=4, blocking=false
auto depthQueue = device.getOutputQueue("depth", 4, false);

// 8. The render/consume loop
while (running) {
    auto rgbFrame   = rgbQueue->get<dai::ImgFrame>();  // blocks until frame ready
    auto depthFrame = depthQueue->get<dai::ImgFrame>();

    // rgbFrame->getCvFrame()  → cv::Mat  (8-bit BGR)
    // depthFrame->getData()   → raw uint16 buffer (depth in mm)
    // depthFrame->getWidth(), getHeight() for dimensions
}

Three Things That Trip Everyone Up

1. Linking direction — always source.outputPort.link(dest.inputPort). Never reversed. If you try to link backwards the API will throw at runtime.
2. One XLinkOut per stream — you cannot have two consumers of the same queue. If you want both depth and disparity on the host, create two separate XLinkOut nodes with different stream names.
3. blocking=false, maxSize=4 — this is the right default for a render loop. blocking=true stalls your UI thread if the OAK produces frames faster than you consume them. Use tryGet() instead of get() if you want non-blocking poll.

The oak-explorer Learning Plan

Six stages — one growing application

Everything is built into a single ImGui application called oak-explorer. Each stage adds a panel or feature. Nothing gets thrown away.

1
Foundation CMake project, ImGui + OpenGL window, depthai-core linked. Device connect/disconnect detection. Understand what the pipeline IS before using it. This stage produces a working window that shows device status.
2
Camera Streams RGB and mono camera nodes. Understand sources, queues, and frame packets. Display live frames in ImGui as GPU textures. Know exactly what data you're receiving and why.
3
Depth & Stereo StereoDepth node, disparity vs. depth, confidence maps. Visualize depth as a heatmap. Understand the math: Z = (f × B) / disparity — focal length, baseline, disparity.
4
Spatial Data SpatialLocationCalculator, 3D point extraction, coordinate systems. Click a pixel in the RGB view, get its XYZ in world space. Overlay live measurements.
5
CUDA Acceleration Copy frames to GPU memory, write custom CUDA kernels (custom colormap, edge filter, point cloud builder). Zero-copy OpenGL interop for display. GTX 3080 starts earning its keep.
6
Neural Inference Deploy a model on the OAK's VPU (MobileNet, YOLO), receive detections, overlay on RGB. Optionally run a second model on the GTX 3080 for comparison or complementary inference.
